

■VPC(仮想プライベートクラウド)
AWS アカウント専用の仮想ネットワーク
VPC は、AWS クラウドの他の仮想ネットワークから論理的に切り離されている
論理的に隔離されている定義済みの仮想ネットワーク内で AWS リソースを起動できる
 
 
■サブネット
各サブネットが完全に 1 つのアベイラビリティーゾーン内に含まれている必要有
※個別のアベイラビリティーゾーンで AWS リソースを起動することにより、1 つのアベイラビリティーゾーンで発生した障害からアプリケーションを保護できる
 
 
■ルートテーブル
データパケットの送信先を決定するためのルールセットを保持するデータ構造(ネットワークトラフィックが適切な宛先に届くように設定される)
*メインルートテーブル：VPC を作成するときに、メインルートテーブルが自動的に割り当てられる
※サブネットに明示的なルーティングテーブルが関連付けられていない場合、デフォルトではメインのルーティングテーブルが使用される
*エントリ：どのネットワークトラフィックがどの宛先にルーティングされるかを定義するルール
 
 
■インターネットゲートウェイ
VPC内のリソースがインターネットと通信できるようにするためのゲートウェイ
*ゲートウェイ：異なるネットワーク同士を接続し、通信を中継する役割を果たす機器や仕組み
 
 
■Egress Only インターネットゲートウェイ
IPv6アドレスを使用するリソースに対してアウトバウンドインターネットアクセスを提供するゲートウェイ
＝IPv4トラフィックには使用されない
名前が示す通り、インバウンドトラフィックは許可しない
 
 
■キャリアゲートウェイ
主にAWS Wavelengthとの連携で使用されるゲートウェイ
 
 
■DHCP オプションセット
VPC内のインスタンスに割り当てるDHCPオプションを定義する設定
インスタンスが起動する際に自動的に受け取るネットワーク構成パラメータを指定可能
*DHCP：ネットワークに接続されるデバイスにIPアドレスやその他のネットワーク設定情報を自動的に割り当てるプロトコル
*domain-name：VPC内のインスタンスに割り当てられるドメイン名(ex.example.com)
*domain-name-servers：インスタンスが指定するDNSサーバのIPアドレスを指定(AmazonProvidedDNSの指定も可能)
 
 
■Elastic IP
静的なパブリックIPv4アドレス
VPC内のリソースがインターネットと直接通信可能になる
*静的IPアドレス：ネットワーク管理者により手動で設定される固定のIPアドレス
*動的IPアドレス：ネットワークに接続されたデバイスに自動的に割り当てられる一時的なIPアドレス
 
 
■マネージドプレフィックスリスト
VPC環境でIPアドレスの範囲(CIDRブロック)をグループ化して管理するためのサービス
複数のCIDRブロックを一つのリストにまとめることが可能
 
 
■エンドポイント
*ネットワークのコンテキストでのエンドポイント：通信の起点または終点となるデバイスやノード
VPCにおけるエンドポイントは、VPC内のリソースがインタ—ネットを経由せずに他のサービスにアクセスできるようにするための仮想デバイス
 
*VPCインターフェイスエンドポイント：
AWS PrivateLinkを使用してAWSサービスにプライベートに接続するためのエンドポイント
ENI(Elastic Network Interface)を使用して、指定したサブネットにプライベートIPアドレスをもつエンドポイントを作成する
*AWS PrivateLink：インターネットを経由せずにAWSのマネージドサービスに安全にプライベート接続できるサービス
あたかも自身のVPC内にS3やDynamoDBのようなサービスが存在するかのようにプライベートにアクセスできる
 
*VPCゲートウェイエンドポイント：
特定のAWSサービス(S3/DynamoDBのみ利用可能)にプライベートに接続するためのエンドポイント
VPCルートテーブルにエントリを追加し、特定のプレフィックスリストへのトラフィックをAWSサービス(S3/DynamoDB)にルーティングする
 
 
■エンドポイントサービス
AWS PrivateLinkを使用して、ユーザーが自身のVPC内でホストするサービスを他のVPCやオンプレミス環境からプライベートにアクセスできるようにする仕組み
 
 
■NATゲートウェイ
プライベートサブネット内のリソースがインターネット上のリソースにアクセスする際に、プライベートIPアドレスをインターネット上で有効なパブリックIPアドレスに変換するためのサービス
インターネットからプライベートサブネット内のリソースへの逆のトラフィックは防ぎつつ、アウトバウンド接続を可能にする
 
 
■ピアリング接続
異なるVPC間でプライベートな通信(インターネットやVPNを経由しない)を可能にするAWSのネットワーキングサービス
 
 
■ネットワークACL
VPC内のサブネットに対して、トラフィックの許可または拒否を制御するためのセキュリティ機能
サブネット単位で、インバウンド・アウトバウンドトラフィックのフィルタリングを行う
 
 
■セキュリティグループ
VPC内のインスタンスに対して、トラフィックの許可または拒否を制御するための仮想ファイアウォール
インスタンス単位で、インバウンド・アウトバウンドトラフィックのフィルタリングを行う
 
 
◆AWS Route53 Resolver DNS Firewall
 
■ルールグループ
ドメイン名ベースのトラフィックを制御するためのルールの集合体
特定のドメイン名に対するDNSクエリを許可またはブロックする
 
 
■ドメインリスト
特定のドメイン名(FQDN)をリスト化したもの
 
 
◆AWS Network Firewall
 
■ファイアウォール
VPC内でトラフィックを検査および制御するためのマネージドファイアウォールサービス
インターネットや他のVPCとの通信を監視し、ルールに基づいて許可または拒否することが可能
ステートフルおよびステートレスの設定が可能
 
 
■ファイアウォールポリシー
ファイアウォールルールのセットを定義し、適用するための設定
特定のトラフィックパターンに対するアクションを一元管理できる
 
 
■ルールグループ
ファイアウォールポリシー内で使用されるルールの集合体
ステートフルグループとステートレスグループの二種類があり、それぞれ異なるトラフィック管理方法となる
ステートフルグループ：トラフィックの状態を追跡し、トラフィックフローに基づいてルールを適用
ステートレスグループ：個々のパケットを独立して評価し、ルールを適用
 
 
■TLS検査設定
暗号化されたトラフィックを復号化し、トラフィック内容を検査するための設定
これにより、暗号化された通信も含めてネットワークトラフィックの詳細な監視が可能となる
 
 
■リソースグループ
AWS Network Firewall関連のリソースを一元管理するためのグループ
これにより、設定や運用が簡素化できる
 
 
◆仮想プライベートネットワーク (VPN)
 
■カスタマーゲートウェイ
オンプレミスネットワークまたはリモートネットワークとAWS VPCを接続するためのAWS側のリソース
オンプレミス側のVPNエンドポイントして機能し、AWS側の仮想プライベートゲートウェイやトランジットゲートウェイと接続する
※VPN接続を確立するために使用される(オンプレミスネットワークとAWS VPC間でセキュアな通信が可能)
 
 
■仮想プライベートゲートウェイ
AWS VPCとオンプレミスネットワークを接続するためのAWS側のVPNエンドポイント
VPCにアタッチされ、カスタマーゲートウェイと連携してSite-to-Site VPN接続を確立する
 
 
■Site-to-Site VPN 接続
AWS VPCとオンプレミスネットワーク間でIPsec VPNトンネルを作成し、セキュアな通信を確立するための接続方式（インターネット経由）
AWS側で仮想プライベートゲートウェイまたはトランジットゲートウェイを使用し、オンプレミス側ではカスタマーゲートウェイデバイスを設定する
この接続は、データの暗号化と認証を提供する
 
 
■クライアント VPN エンドポイント
リモートユーザーがAWSリソースやオンプレミスネットワークにセキュアにアクセスするためのエンドポイント
OpenVPNプロトコルを使用し、ユーザー認証とアクセス制御を提供する
 
 
◆AWS Verified Access
VPC内のリソースへのアクセスを厳密に制御するためのサービス
 
■Verified Access インスタンス
Verified Accessの設定やポリシーが適用される対象のリソースやアプリケーション
これらのインスタンスは、ユーザーやデバイスからアクセスリクエストを評価し、事前定義されたセキュリティポリシーに基づいてアクセスが許可または拒否される
 
 
■Verified Access 信頼プロバイダー
ユーザーのアイデンティティやデバイスのセキュリティ状態を検証するための外部サービス
サードパーティーのセキュリティシステムと統合することで、信頼プロバイダーからのセキュリティシグナルを使用してアクセスリクエストを評価する
 
 
■Verified Access グループ
アクセス制御を効率的に管理するために使用されるエンティティの集合体
ユーザーやデバイスをグループに分類し、グループごとに異なるアクセスポリシーを適用することが可能となる
 
 
■Verified Access エンドポイント
AWS Verified Accessを介して保護されるリソースやアプリケーションへの接続ポイント
 
 
◆Transit Gateway
 
■Transit Gateway
複数のVPCやオンプレミスネットワークを相互接続するためのネットワークハブ
複雑なVPCピアリング(1対1)の代わりに一元的な接続管理が可能となる
 
 
■アタッチメント
Transit GatewayとVPC,VPN,Direct Connectなどのネットワーク間の接続ポイント
各アタッチメントは特定のVPCやオンプレミスネットワークに関連付けられ、これにより異なるネットワーク間でのトラフィックの流れを管理できる
アタッチメントを作成する際は、接続するVPCやサブネットを指定する
 
 
■ポリシーテーブル
Transit Gateway内でのトラフィックの許可や拒否を制御するための設定
 
 
■ルートテーブル
ネットワークトラフィックの経路を提起するための表
各アタッチメントがどのように通信するかを決定し、複数のVPCやオンプレミスネットワーク間のトラフィックを適切にルーティングする
 
 
■マルチキャストドメイン
複数の受信者に対して同時にデータを配信するための設定
 
 
◆トラフィックミラーリング
VPC内のインスタンスから送信および受信されるネットワークトラフィックをキャプチャし、それを別の宛先(ex.ネットワークパケット解析ツールやセキュリティ監視ツール)に送信する機能
 
■トラフィックミラーセッション
ミラーリングするトラフィックの送信元と送信先を定義する設定
セッションはフィルターを使用して、特定のトラフィックのみをミラーリングする
 
 
■トラフィックミラーターゲット
ミラーリングされたトラフィックの送信先
 
 
■トラフィックミラーフィルタ
ミラーリングするトラフィックを選別するための条件
フィルターは、送信元や受信先のIPアドレス、ポート番号、プロトコルなど基にトラフィックを選別する
 
 
◆Amazon VPC Lattice
複数のVPCやAWSアカウントにまたがるサービスを接続、監視、保護するための完全マネージド型アプリケーションネットワーキングサービス
複数のVPC間でのサービス間の通信を容易に設定し、管理することが可能となる
 
■サービスネットワーク
特定のサービス間での通信を制御するための仮想ネットワーク
VPCとサービスを関連付けることで、その範囲内での通信が可能となる
 
 
■サービス
リスナーやターゲットグループを指定し、リクエストを受け取って指定のターゲットにルーティングする役割を持つ
 
 
■ターゲットグループ
トラフィックを特定のバックエンドリソースにルーティングする
*ターゲットの種類：EC2インスタンス、IPアドレス、Lambda関数等
 
 
■ネットワークマネージャー
複数のAWSアカウントやリージョンにまたがるネットワークリソースを一元管理するためのツール
 
 
■Cloud WAN
複数のオンプレミスネットワークやVPC間の接続を統合的に管理するサービス
 
 
■VPC IPアドレスマネージャー(IPAM)
VPC内のIPアドレス管理を自動化・効率化するツール
IPアドレスの重複や枯渇を防ぎ、ネットワーク設計の最適化が可能


◆インスタンス概要
 
■インスタンスID
EC2インスタンスを一意に識別するためのユニークな識別子
 
 
■パブリックIPv4アドレス
インターネット上から直接アクセス可能なグローバルIPアドレス
外部からの通信に使用される
 
 
■プライベートIPv4アドレス
VPC内で使用されるローカルネットワーク用のIPアドレス
 
 
■IPv6アドレス
IPv6対応のインスタンスに割り当てられるアドレス、インターネットやVPC内での通信に使用される
 
 
■インスタンスの状態
インスタンスの現在の稼働状態を示す
 
 
■パブリックIPv4 DNS
パブリックIPv4アドレスに対応するDNS名(ドメイン形式でアクセス可能)
インターネット上でインスタンスにアクセスするときに使用される
動的なパブリックIPアドレスを持つインスタンスでも、DNS名を使うことで安定した接続が可能
 
 
■ホスト名のタイプ
EC2インスタンスのホスト名の形式を指定する設定
ホスト名はインスタンスを識別するための名前で、EC2インスタンス起動時に自動的に割り当てられる
1.リソースベースの名前
インスタンスIDを含む形式のホスト名
2.IPベースの名前
IPアドレスを含む形式のホスト名
 
 
■プライベートIP DNS名(IPv4のみ)
プライベートIPアドレスに関連付けられたDNS名
プライベートIPアドレスの代わりにDNS名で他のインスタンスからアクセスできる
VPC内でのみ有効、インタネット上からはアクセス不可
 
 
■プライベートリソースの DNS 名に応答
VPC内のプライベートDNSホストゾーンでホストされたプライベートDNSレコードに対して、
インスタンスが応答するかどうかを制御する
 
 
■インスタンスタイプ
EC2インスタンスの仮想ハードウェアの構成を定義する
CPU,メモリ,ストレージ,ネットワークパフォーマンス等のリソースの組み合わせが異なる様々なインスタンスタイプが存在する
例）
m6.large
m:インスタンスファミリー
*インスタンスファミリー：同じような特性や用途に最適化された一連のインスタンスのグループ
6:世代(数字が大きいほど性能が向上し、コストも上がることが多い)
large:インスタンスサイズ
*インスタンスサイズ：インスタンスがもつリソースの大きさや性能を示す指標(サイズが大きくなるにつれて、CPU、メモリ、ストレージなどのリソースが増加する)
 
■Elastic IP アドレス(EIP)
AWSで提供される固定のパブリックIPv4アドレス
インスタンスが停止、再起動されても変更されず、特定のEC2インスタンスにアタッチして使用できる
 
 
■自動的に割り当てられた IP アドレス
インスタンス起動時にAWSが自動的に割り当てるパブリックIPv4アドレス
Elastic IP アドレス(EIP)と異なり、このアドレスはインスタンスが停止すると解放され、再起動時には異なるIPアドレスが割り当てられる
 
 
■VPC ID
インスタンスが所属するVPCのユニークな識別子
 
 
■AWS Compute Optimizer の検出結果
インスタンスの使用状況に基づいて最適なインスタンスタイプやサイズを提案するサービス
 
 
■IAM ロール
インスタンスにアタッチされているIAMロール
 
 
■サブネット ID
VPC内でインスタンスが配置されるサブネットを一意に識別するID
 
 
■Auto Scaling グループ名
EC2インスタンスのスケーリングを管理するAuto Scaling グループの名前
指定された条件に基づいてインスタンスの数を自動的に増減させる機能を持つ
 
 
■IMDSv2
EC2インスタンスが自身のメタデータにアクセスするためのセキュアなインターフェース
IMDSv1と比べてセキュリティが強化されている
 
 
■メタデータ
MetadataOptions.HttpTokens
上記はEC2インスタンスが自身のメタデータにアクセスする際に、HTTPヘッダーでAWSセキュリティトークンを要求するか否かを決定するオプション
*メタデータ：データに関する情報を格納したデータ(HTTPやIMDSv2等でアクセス可能)
EC2インスタンスは、インスタンスの詳細情報、ネットワーク設定、セキュリティグループ情報などの情報を提供するメタデータサーバ—を持っている
例）
インスタンスID
インスタンスタイプ
リージョン
パブリックIPアドレス
プライベートIPアドレス
セキュリティグループ
ユーザーデータ
*HTTPヘッダー：Webサーバーとクライアントの間で、HTTPメッセージに関する追加情報を送受信するための情報群
*HTTPメッセージ：WebクライアントとWebサーバーの間でやり取りされるデータ(HTTPリクエストとHTTPレスポンスの二種類に分けられる)
*IMDSv1(Instance Metadata Service)：EC2インスタンスに特化したサービス、HTTPを使用してメタデータを取得する
*IMDSv2(Instance Metadata Service)：EC2インスタンスに特化したサービス、HTTPSを使用してメタデータを取得する、デフォルトで443番ポートを使用する(Requiredでないと使用不可)
 
MetadataOptions.HttpEndpoint
IMDSv2が443ポートにアクセスできるようにするために必要なオプション(443ポートへのアクセスを可能にするかどうかを制御するためのオプション)
disabledの場合、IMDSv2を使用してインスタンスメタデータにアクセスできなくなる
disabledの場合、IMDSv2を使用するためのネットワークポートが開かれてない(HTTPS)
つまり、disabledでもHTTPやHTTPを使用したIMDSv1は使用できるということ(enabledの場合、HTTP、IMDSv1、IMDSv2も使用可能)
*HTTPSでの通信：受信側(サーバ)は443ポートを使用する、送信側(クライアント)は443ポートやエフェメラルポート(一時的に使用されるポート)を使用する(1024-65535)ことが多いが特に決まりはない？
 
 
■インスタンス ARN
EC2インスタンスを一意に識別するためのグローバルな識別子
 
 
◆インスタンスの詳細
 
■プラットフォーム
インスタンスが動作するOSの種類
 
 
■AMI ID
インスタンスのベースとなるAmazonマシンイメージ(AMI)の識別子
 
 
■モニタリング
インスタンスのパフォーマンスデータ(CPU使用率・ディスクI/Oなど)を収集するためのモニタリング設定
基本モニタリングと詳細モニタリングがある(CloudWatchの機能)
 
 
■プラットフォームの詳細
プラットフォームに関する追加情報(OSのバージョン等)
 
 
■AMI 名
インスタンスが作成された元のAMIの名前
 
 
■終了保護
誤ってインスタンスを終了することを防ぐ機能
手動で終了する操作がブロックされる
 
 
■停止保護
誤ってインスタンスを停止することを防ぐ機能
手動で停止する操作がブロックされる
 
 
■起動時刻
インスタンスが最後に起動された日時
 
 
■AMI の場所
インスタンスを作成するために使用されたAMIの保存場所
AMIがどのリージョンに保存されているかを示す
 
 
■インスタンスの自動復旧
ハードウェア障害やソフトウェア障害が発生した際に、インスタンスを自動的に復旧させる機能
AWSは同じインスタンスIDを保持しつつ、新しいハードウェアで再起動する
 
 
■ライフサイクル
インスタンスの状態を示す
 
 
■停止 - 休止動作
インスタンスが停止されたときにメモリ内容を保持したままの状態にする「休止モード」を有効にするかどうかの設定
 
 
■AMI 起動インデックス
Auto Scalingや他のマルチインスタンス起動プロセスで、インスタンスがどの順序で起動されたかを示すインデックス番号
 
 
■起動時に割り当てられたキーペア
インスタンスの初回起動時にSSHアクセス用に割り当てられたキーペアの名前
このキーペアを使用してインスタンスにセキュアにアクセスができる
 
 
■状態遷移の理由
インスタンスの状態(起動・停止・終了など)が遷移した理由
 
 
■クレジット仕様
T2等のバースト可能なインスタンスタイプにおいて、クレジットの取得と消費方法を示す
クレジットは、通常のパフォーマンスが不足する際に、追加のCPUリソースを使用するためのポイント
 
 
■カーネル ID
インスタンスで使用されるカーネルの識別子
特定のカーネルバージョンが必要な場合に、このIDを指定してインスタンスを起動する
 
 
■状態遷移メッセージ
インスタンスの状態遷移に関連する追加メッセージ
 
 
■使用オペレーション
インスタンスが実行しているオペレーションや作業内容を示す情報
 
 
■RAM ディスク ID
インスタンスが使用するRAMディスクの識別子
 
 
■所有者
インスタンスの所有者を示す
通常、インスタンスを作成したAWSアカウントの情報が表示される
 
 
■Enclaves のサポート
AWS Nitro Envlavesのサポートが有効かどうかを示す
Nitro Envlavesは、高度に分離された環境を提供し、機密データの処理やセキュリティを強化するために使用される
 
 
■ブートモード
インスタンスのブートモード(Legacy BIOS,UEFI)を指定する
 
 
■現在のインスタンスのブートモード
現在のインスタンスがどのブートモードで実行されているかを示す
 
 
■インスタンスメタデータのタグを許可する
インスタンスメタデータにタグを設定できるかどうかを示す
 
 
■ゲスト OS ホスト名として RBN を使用
インスタンスのゲストOSで、AWSが提供するリバースビジネスネーム(RBN)をホスト名として使用するかどうかを設定する
 
 
■RBN DNS ホスト名 IPv4 に応答
インスタンスがリバースビジネスネーム(RBN)で設定されたDNSホスト名に対して、IPv4アドレスで応答するかどうかを示す
 
 
◆ホストとプレイスメントグループ
 
■ホストID
専有ホスト上で実行されているインスタンスに対して割り当てられるホストの識別子
専有ホスト：特定の物理サーバーを単一のAWSアカウントで専有して使用するためのオプション
 
 
■アフィニティ
特定のインスタンスを特定の専有ホストに固定するための設定
アフィニティを設定することでインスタンスが起動・再起動する際に同じ専有ホストで動作し続けることが保証される
 
 
■プレイスメントグループ
インスタンス間のネットワークパフォーマンスを最適化するために、物理的に近い場所にインスタンスを配置するグループ設定
 
クラスタープレイスメントグループ：
スプレッドプレイスメントグループ：
パーティションプレイスメントグループ：
 
 
■ホストリソースグループ名
AWS Resource Groupsで管理されている、専有ホストを含むリソースのグループ名
専有ホストをグループ化し、統一された管理が可能となる
 
 
■テナンシー
インスタンスが実行される環境を示す設定
3つのタイプが存在する
共有(default)
専有ホスト
専有インスタンス
 
 
■プレイスメントグループ ID
インスタンスが属しているプレイスメントグループの識別子
プレイスメントグループを識別し、管理する際に使用される
 
 
■仮想化タイプ
インスタンスの仮想化方式を示す
HVM：ハードウェア仮想化
PV:パラバーチャライゼーション
 
 
■予約
リザーブドインスタンスとして予約されたリソースの詳細
リザーブドインスタンスは1年または3年単位で使用量を予約することでオンデマンド料金よりもコストを削減できる
 
 
■パーティション番号
パーティションプレイスメントグループで使用される、インスタンスが属するパーティションの番号
 
 
■vCPU の数
インスタンスに割り当てられている仮想CPUの数を示す
vCPUはインスタンスの計算能力を表す
 
 
◆キャパシティーの予約
 
■キャパシティーの予約 ID
特定のキャパシティ予約を識別するための識別子
 
 
■キャパシティ予約
特定のインスタンスタイプのEC2インスタンスを起動可能状態になるための権利の予約
ゾーンリザーブドインスタンスの場合、同時にキャパシティ予約も可能となる
 
 
■キャパシティーの予約の設定
キャパシティ予約の詳細設定を示す
どのインスタンスが予約されたキャパシティを使用できるかを制御する
 
◆アクセラレーター
 
■Elastic Graphics ID
インスタンスにアタッチされたElastic Graphicsアクセラレータの識別子
Elastic Graphicsはインスタンスに追加のGPUリソースを提供し、グラフィック処理を強化する
 
 
 
 
■ユーザーデータ
CloudFormationにてユーザーデータを変更してスタック更新をかけるとEC2に再起動がかかるので要注意
 
 
■タグ情報
タグは10個を超えると2ページ目に追加される
 
 
■コア数とスレッド数
*コア数：CPU内で同時に処理できる命令の数
*スレッド数：一つのコアで同時に処理できる命令の数
コアあたりの有効なスレッド数は、各コアで処理できるスレッドの最大数を設定するもの(1 or 2)
1つのスレッド：各コアで1つのスレッドしか処理できない
2つのスレッド(ハイパースレッディング)：各コアで2つのスレッドを処理することができる
*ハイパースレッディング技術：CPU一つに対して、論理コアを2つ割り当て、あたかも2つの論理コアが独立して動作しているように見せる技術
*VCPU:物理CPUのコアとスレッドを共有することで動作する、物理CPUと比較して性能が劣る可能性有
